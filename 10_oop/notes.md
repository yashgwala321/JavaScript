TOPIC: JavaScript OOP (Object Oriented Programming)

1) OOP kya hota hai?
   - OOP ek programming style hai

   - Object ke andar data + functions dono hote hain

2) Object kya hota hai?
   - Object = properties + methods ka collection
   - Properties → data (username, loginCount, etc.)
   - Methods → functions (getUserDetails, greeting, etc.)

3) Object Literal:
   - Direct object banana
   - Small use-case ke liye best
   - Example: const user = { name, age, function }

4) Constructor Function:
   - Same structure ke multiple objects banane ke liye
   - Function capital letter se start hota hai (good practice)
   - new keyword se call hota hai
   - this keyword current object ko refer karta hai

5) new keyword ka kaam:
   - Naya empty object create karta hai
   - this ko us object se bind karta hai
   - Constructor function ko automatically return karta hai

6) this keyword:
   - this current object ko point karta hai
   - Object ke andar this → us object ko refer karta hai
   - Global scope me this → window object hota hai

7) Method kya hota hai?
   - Object ke andar bana function
   - Object ke data ko access kar sakta hai using this

8) Instances:
   - Constructor function se bane hue objects
   - Example: userOne, userTwo
   - Har instance apna alag data rakhta hai

9) Constructor property:
   - Batata hai object kis constructor se bana hai
   - Debugging aur checking ke kaam aata hai

10) Why use OOP?
    - Code reusable hota hai
    - Structure clear hota hai
    - Large projects me manage karna easy hota hai
    - Real-world logic ko easily represent kar sakte hain

11) Parts of OOP in JavaScript:
    - Object Literal
    - Constructor Function
    - Prototypes
    - Classes
    - Instances (new, this)

12) 4 Pillars of OOP:
    - Abstraction → Sirf important cheez dikhana
    - Encapsulation → Data ko secure rakhna
    - Inheritance → Dusri class ke features use karna
    - Polymorphism → Same method, different behavior
